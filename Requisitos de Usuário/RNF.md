# 1. Requisitos Não Funcionais

<p align="justify">A <i>Tabela 2</i> a seguir contém os <b>Requisitos Não Funcionais (RNF)</b> elicitados utizando a técnica de Brainstorm .</p>

| ID   | Requisito | Prioridade | Requisitos Relacionados |
| :--: | :-------------------------------------------------------------------------------------------------------------------------: | :--------: | :----------------------: |
| RNF01 | O sistema deve autenticar usuários por meio de credenciais (e-mail e senha), garantindo que apenas usuários cadastrados acessem a plataforma. | Alta | RF01 |
| RNF02 | O sistema deve responder às principais operações do usuário em até 2 segundos, exceto upload de arquivos. | Alta | RF01, RF02, RF03, RF04, RF05 |
| RNF03 | O sistema deve estar disponível durante 99% do tempo, desconsiderando períodos programados de manutenção. | Média | Todos os RF |
| RNF04 | O sistema deve possuir interface responsiva, permitindo sua utilização em computadores, tablets e smartphones. | Média | Todos os RF |
| RNF05 | O sistema deve armazenar os arquivos compartilhados de forma segura, permitindo acesso apenas aos membros do grupo correspondente. | Alta | RF06 |
| RNF06 | O sistema deve utilizar conexão segura (HTTPS) para transmissão de dados entre cliente e servidor. | Alta | RF01, RF05, RF06 |
